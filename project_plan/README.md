项目规划总览（60 天 IRL 主线）
================================

目标（V0 / 假数据阶段）
-----------------------

- 基于 `data/isabella_irl_3d_clean` 的 60 天 nightly 假数据，跑通一条端到端主线：
  - 日级多模态数据 → 日级特征、时间序列表示；
  - 时间序列表示 → 个体人格/偏好向量；
  - 与预设 Big Five / BFI 报告做相关性评估；
  - 提供一个可替换数据源的干净接口，后续用真实 Town / 真人数据替换时，模型代码尽量不动。

- 暂时**不强求**：
  - 完整的多模态 GT 标注与权重学习；
  - 工业级的 IRL 理论完备性。
  - 这些可以在 V1/V2 迭代。

阶段划分（面向 3 人协作）
------------------------

- Phase 1（P1）：数据接口 & 基础特征
  - Owner-Data（A）主负责。
  - 统一 60 天数据加载接口，完成日级基础特征抽取。

- Phase 2（P2）：时间序列表示 & 人格回归
  - Owner-Sequence（B）主负责。
  - 实现滑动窗口统计 + 衰减机制 + BiLSTM/GRU+注意力，先完成“轨迹 → Big Five 回归”。

- Phase 3（P3）：IRL / 偏好建模 MVP
  - Owner-IRL（C）主负责。
  - 在 P2 的状态表示上，做一个最小可用的 IRL / 逆回归实验，并和人格/BFI 做对齐评估。

角色与分工文件
--------------

- `roles_and_ownership.md`  
  定义三类 owner 的职责边界、交付物与接口。

- `timeline_and_milestones.md`  
  给出按周划分的排期和阶段 Milestone，默认以 3 周为一期（可按实际节奏拉长）。

- `task_board.md`  
  按 ToDo / InProgress / Done 维护任务列表，方便在代码层面同步「当前做到哪」。

后续真实数据接入方式
--------------------

- 真实 Town / 真人数据准备好后，**优先保证以下不变**：
  - P1 的 `DatasetLoader` 接口；
  - P2/P3 的模型训练脚本与评价指标。

- 为新数据源单独写 loader / 清洗脚本，将其适配到同样的「日级特征矩阵 + 标签/奖励」格式，
  直接复用当前训练与评估代码，做对比、微调与论文实验。
